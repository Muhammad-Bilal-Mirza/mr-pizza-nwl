# Local Development Setup

This guide explains how to run the Mr. Pizza application locally with all images loading properly.

## Prerequisites

You'll need:
- **Node.js** (v18+) - [Download](https://nodejs.org/)
- **pnpm** - Install with: `npm install -g pnpm`
- **Git** (already have it if you cloned this repo)

## Quick Start (Recommended for Development)

The fastest way to get started is using the mock backend, which requires no external services:

### 1. Install dependencies

```bash
cd src/frontend
pnpm install
```

### 2. Start the development server

```bash
pnpm dev
```

The app will start at `http://localhost:5173/` (or another port if 5173 is in use).

### 3. View the application

- Open http://localhost:5173/ in your browser
- You'll see the pizza menu with all images loading from `/public/assets/generated/`
- The mock backend provides sample menu data
- All pizza images are served locally from the `public/assets/generated/` directory

## Image Loading Configuration

Images are configured in multiple places to ensure proper loading:

1. **App.tsx** - Maps pizza names to local image paths
2. **DEFAULT_STATIC_MENU** - Includes local image URLs
3. **Vite Config** - Public assets are served from `/` base URL

The Vite server automatically serves everything in `public/` folder at the root URL.

## Full Stack Testing (with Local IC Replica)

If you want to test with a real backend canister:

### 1. Install dfx

```bash
sh -ci "$(curl -fsSL https://sdk.dfinity.org/install.sh)"
```

### 2. (Optional) Update environment

Create/update `src/frontend/.env.local` with:

```
VITE_USE_MOCK=false
DFX_NETWORK=local
```

### 3. Start IC replica

```bash
dfx start --clean
```

(in a separate terminal)

### 4. Deploy backend

```bash
dfx deploy backend
```

### 5. Get your canister ID

```bash
dfx canister id backend
```

### 6. Update configuration

Update `src/frontend/env.json` with your local canister ID:

```json
{
  "backend_host": "http://localhost:4943",
  "backend_canister_id": "your-canister-id-here",
  "project_id": "0000000-0000-0000-0000-00000000000",
  "ii_derivation_origin": ""
}
```

### 7. Run the app

```bash
cd src/frontend
pnpm dev
```

## Troubleshooting

### Images not loading?

1. **Check browser console** - Look for 404 errors
2. **Verify file exists** - Run `ls src/frontend/public/assets/generated/`
3. **Check Vite logs** - Look for "public" in the dev server output
4. **Clear cache** - Ctrl+Shift+Del (or Cmd+Shift+Del) to do a hard refresh

### Development server won't start?

```bash
# Clear node_modules and reinstall
cd src/frontend
rm -rf node_modules pnpm-lock.yaml
pnpm install
pnpm dev
```

### Port 5173 already in use?

Vite will automatically try the next available port. Check your terminal output for the actual URL.
